# Design: WealthTrack — 個人資產追蹤 App

Generated by /office-hours on 2026-04-05
Branch: main

## Problem Statement

使用者目前用 Google Sheets 追蹤個人資產。Google Sheets 可以透過 GOOGLEFINANCE 函數自動更新股價，但增刪改查操作不方便：每次新增資產要找對的欄位、填入公式、確保格式正確。需要一個操作體驗更好的手機 app，同時加上即時價格更新和分析功能。

資產類型涵蓋：
- 現金：台幣（依銀行區分）、外幣（依銀行+幣別區分）
- 股票：台股（依股票代號）、美股（依股票代號）
- 基金：國內基金（依基金代號）
- 加密貨幣（依代號區分）

## What Makes This Cool

不只是 Google Sheets 的美化版。這個 app 會有：
1. **即時價格更新** — 開 app 就能看到當前淨值，不用等 Sheets 刷新
2. **視覺化分析** — 圓餅圖看資產配置、折線圖看走勢、報酬率計算
3. **台灣在地化** — 所有資產自動換算台幣總值、支援台股 TWSE、外幣匯率
4. **壓力測試** — what-if scenarios（「如果美元跌 5%，我的淨值變多少？」）
5. **全 Flutter 練功作品** — 從資料庫、API 整合、狀態管理到圖表，完整技術棧

## Constraints

- 使用者前端經驗不多，Flutter 是新學的
- 主要自己用，也會給朋友家人用，最後放作品集
- 台灣市場：銀行沒有 Plaid-style API，資產手動輸入
- 資料存本地（手機端），v1 不需要後端
- 需要免費或低成本的價格 API

## Premises

1. 資產數據手動輸入（不需要自動同步銀行帳戶），價格由 API 自動更新
2. 單人使用的 app，不需要用戶註冊/登入系統（至少 v1）
3. 數據存在本地（手機上），不需要後端服務器
4. 需要的價格 API：台股（TWSE）、美股、加密貨幣、外幣匯率
5. 先做 Flutter 手機版，Web 版之後再說

## Cross-Model Perspective

獨立 Claude subagent 的冷讀分析：

- **最酷的版本：** 加上「台灣本地化智慧層」，包括台股除權息日處理、外幣匯率交叉換算（所有資產用台幣計價）、壓力測試面板、以及加密備份到自己的雲端資料夾（仍然無後端但不怕手機遺失）
- **關鍵洞察：** 使用者的興奮點在「產品深度」（即時+分析+完整CRUD），不只是修好資料輸入
- **推薦參考：** Portfolio Performance（Java 投資組合追蹤器）的資料模型設計，和 Ghostfolio 的產品邏輯
- **週末原型：** 只做「美股 + 台幣現金」一條垂直切片，證明整個流程可以跑通

## Approaches Considered

### Approach A: 「垂直切片」漸進式（已選擇）

先做一個資產類型（美股+台幣現金）跑通全流程（CRUD + 即時價格 + 圖表），再逐步加入其他資產類型。每一步都有可用的產品。

- Effort: M（每個資產類型約 1-2 天）
- Risk: Low
- Pros: 每步都有成果、學習曲線平緩、不會被卡住
- Cons: 可能在加入新資產類型時需要重構資料模型

### Approach B: 「全資產架構」一步到位

先設計完整的資料模型和架構，一次支援所有資產類型，再填充 UI。

- Effort: L
- Risk: Med
- Pros: 架構乾淨不需重構
- Cons: 要等較久才看得到結果，前期容易過度設計

### Approach C: 「參考開源」加速版

研究 Ghostfolio/Portfolio Performance 的資料模型，用它們的設計作為基礎。

- Effort: M
- Risk: Low
- Pros: 資料模型已被驗證
- Cons: 可能被別人的設計限制住，少了「從零設計」的學習

## Recommended Approach

**Approach A — 垂直切片漸進式**，同時參考 Ghostfolio 的資料模型精神。

原因：學習的最大價值在於「每一步都有可用的東西」，而不是「設計了三天還看不到畫面」。Flutter 新手最怕的是前期投入太多時間在架構設計上，結果還沒看到任何視覺結果就失去動力。

### 開發階段

**Phase 1: 基礎建設（1-2 天）**
- Flutter 專案初始化 + 專案結構
- 本地資料庫設定（SQLite via drift/sqflite）
- 基礎資料模型設計（Asset 基類，可擴展）
- 導航框架（Bottom Navigation：首頁 / 資產列表 / 設定）

**Phase 2: 第一條垂直切片 — 台幣現金（1-2 天）**
- 台幣現金的 CRUD（新增/編輯/刪除/查看）→ 對應需求 4, 5, 6, 7
- 首頁顯示台幣現金總額 → 對應需求 1
- 簡單的銀行分布圓餅圖 → 對應需求 2, 3a
- 列表頁：依銀行分組顯示，點擊可展開細項

**Phase 3: 第二條切片 — 美股（2-3 天）**
- 美股持倉 CRUD（股票代號、數量、買入價格、支援碎股）
- 價格 API 整合：Yahoo Finance (primary)，失敗時 fallback Alpha Vantage，本地快取 15 分鐘
- 美股市值計算 + USD→TWD 匯率換算
- 首頁更新：台幣現金 + 美股 → 對應需求 1, 2
- 資產分布圓餅圖（現金 vs 美股）→ 對應需求 2, 3b
- 股票列表頁：依代號顯示，點擊查看詳細（當前價/成本/報酬率）→ 對應需求 7

**Phase 4: 加入台股（1-2 天）**
- 台股 CRUD + TWSE API 整合
- 即時台股價格更新
- 首頁總資產 = 台幣現金 + 美股(TWD) + 台股

**Phase 5: 外幣 + 基金 + 加密貨幣（2-3 天）**
- 外幣現金 CRUD + 匯率 API → 對應需求 3a-ii
- 國內基金 CRUD（淨值手動輸入，無自動 API）→ 對應需求 3c
  - 基金列表頁：依基金代號顯示，點擊查看詳細（當前淨值/成本/單位數）→ 同股票 UI pattern
- 加密貨幣 CRUD + CoinGecko API → 對應需求 3d
- 完整資產分布圓餅圖 → 對應需求 2

**學習策略：** Flutter 新手同時學 drift + Riverpod + go_router + dio 會有負擔。建議 Phase 1-2 就用 Riverpod（從一開始建立好的狀態管理習慣），但 API 呼叫先用簡單的 http package，Phase 3 再引入 dio。這樣不需要之後大規模重構。

**API 風險管理：** Yahoo Finance 是非官方 API，可能隨時變動。策略：
- 批次查詢（一次傳多個 symbol），減少請求數
- 本地快取 15 分鐘，避免重複請求
- 失敗時 fallback 到 Alpha Vantage（25 req/day free tier）
- v1 上限 20 支美股，確保在 free tier 內
- 如果兩個都失敗，顯示上次快取的價格 + 「價格未更新」標示

**Phase 6: 分析功能（2-3 天）**
- 資產總值 30 天走勢圖（fl_chart LineChart，基於每日淨值快照）← v1 成功標準之一
- 各類資產報酬率計算（額外功能，非原始需求）
- 資產配置趨勢分析（額外功能，非原始需求）

**Phase 7: 進階功能（選做，不在 v1 成功標準內）**
- 壓力測試 what-if 面板
- 加密備份到 Google Drive / iCloud（需要獨立技術調研）
- Widget 首頁小工具顯示淨值
- 深色模式
- 匯率變動影響分析

### 技術棧

| 層級 | 技術 | 原因 |
|------|------|------|
| 框架 | Flutter 3.x | 跨平台、學習目標 |
| 語言 | Dart | Flutter 原生 |
| 本地 DB | drift (moor successor) | 類型安全的 SQLite ORM，比 sqflite 好學 |
| 狀態管理 | Riverpod | 現代、類型安全、社群推薦 |
| 圖表 | fl_chart | 7400+ stars，支援所有需要的圖表類型 |
| HTTP | dio | 比 http 更完整，攔截器/快取支援好 |
| 台股+美股 API | Yahoo Finance (unofficial, primary) — 台股用 `.TW` 後綴（如 `2330.TW`） | 統一 API，一套 code 查兩個市場 |
| 台股 fallback | TWSE OpenAPI (`openapi.twse.com.tw`) | 當 Yahoo 掛了時使用 |
| 美股 fallback | Alpha Vantage (25 req/day free) | 本地快取 15 min，v1 上限 20 支美股 |
| 加密幣 API | CoinGecko (`api.coingecko.com/api/v3`) | 免費 30 req/min，夠用 |
| 匯率 API | ExchangeRate-API (free tier, 1500 req/mo) | 每天更新一次匯率即可 |
| 基金淨值 | v1 手動輸入 | 無穩定免費 API，未來可加爬蟲 |
| 路由 | go_router | Flutter 官方推薦 |

### 資料模型（核心）

```dart
// 基礎資產（sealed class — 編譯器強制 switch 窮舉檢查）
sealed class Asset {
  String id;          // UUID
  String name;        // 顯示名稱
  DateTime createdAt;
  DateTime updatedAt;
}

enum AssetType { cash, stock, fund, crypto }
enum StockMarket { tw, us }

// 現金
class CashAsset extends Asset {
  CurrencyCode currency;  // TWD, USD, JPY...
  String bankName;         // 銀行名稱
  double amount;           // 金額
}

// 股票（shares 用 double 支援美股碎股）
class StockAsset extends Asset {
  String symbol;           // 2330, AAPL
  StockMarket market;      // TW, US
  double shares;           // 持有股數（double 支援碎股）
  double avgCost;          // 平均成本（以交易幣別計）
  double? currentPrice;    // 最新價格（API 更新）
  DateTime? priceUpdatedAt; // 價格拉取時間（過期分級用）
}

// 基金
class FundAsset extends Asset {
  String fundCode;         // 基金代號
  double units;            // 持有單位數
  double avgCost;          // 平均成本
  double? currentNav;      // 即時淨值
  DateTime? navUpdatedAt;  // 淨值更新時間
}

// 加密貨幣
class CryptoAsset extends Asset {
  String symbol;           // BTC, ETH
  double amount;           // 持有數量
  double avgCost;          // 平均成本（以交易幣別計）
  double? currentPrice;
  DateTime? priceUpdatedAt;
}

// 每日淨值快照（keyed to Asia/Taipei local date）
class DailySnapshot {
  String date;         // yyyy-MM-dd (Asia/Taipei)
  double totalValueTWD;
  double cashValue;
  double stockValue;
  double fundValue;
  double cryptoValue;
}

// 匯率（持久化到 DB，歷史可查）
class ExchangeRate {
  String fromCurrency;  // USD, JPY...
  String toCurrency;    // TWD
  double rate;
  DateTime fetchedAt;
}
```

**v1 假設（簡化規則）：**
- 每個股票代號只有一筆持倉（不支援多筆分批買入）
- 不處理公司行動（除權息、股票分割）
- 空資產時顯示引導畫面（「新增你的第一筆資產」）
- 每個資產類型的列表頁和詳細頁使用相同的 UI pattern

**資料庫映射（drift）：**
- 分表設計：`cash_assets`, `stock_assets`, `fund_assets`, `crypto_assets` 各一張表
- 共用欄位（id, name, created_at, updated_at）用 drift mixin 提取，避免重複定義
- `daily_snapshots` 表儲存每日淨值快照
- `exchange_rates` 表持久化匯率，重啟後仍可用
- 總資產查詢：四張表各自 SUM 後加總，資料量小（< 100 筆）不需要 JOIN 或 VIEW

## Design Decisions (v1 defaults)

1. **價格更新頻率：** 每次開 app 或手動下拉刷新時拉一次（best-effort，非即時串流）。盤中盤後同一策略，API 回傳的就是最新可用價格。
2. **歷史淨值：** 每天首次開 app 時記錄一筆淨值快照到本地 DB（走勢圖資料來源）。如果當天沒開 app，該天就沒有資料點，走勢圖跳過。
3. **多裝置同步：** v1 不做。資料模型不預留同步欄位，避免過度設計。
4. **資產交易記錄：** v1 只記當前持倉（quantity + avgCost），不做完整交易紀錄。
5. **成本幣別：** 所有 avgCost 以該資產的原始交易幣別儲存（美股用 USD、台股用 TWD）。顯示報酬率時再用當前匯率換算台幣。
6. **「即時」定義：** 文件中的「即時」指「開 app 時自動拉取最新價格」，非 WebSocket 串流。TWSE 和 Yahoo Finance 提供的是延遲報價（15-20 分鐘），這對個人資產追蹤夠用。
7. **價格過期分級：** < 1h 正常顯示、1-24h 淡色標註「昨天的價格」、> 24h 醒目警告「價格已過期」、從未拉過顯示「等待價格更新」。
8. **API 快取間隔（per source）：** 股票 15 min、加密貨幣 5 min、匯率 24h。間隔內再次請求直接用快取。
9. **專案結構：** Feature-first + Clean Architecture（lib/features/{feature}/data|domain|presentation/）。
10. **狀態管理：** Riverpod with code generation（@riverpod annotation + build_runner）。
11. **測試框架：** flutter_test + mocktail（不需要 code gen 的 mock 方案）。

## NOT in scope (v1)

- 用戶註冊/登入系統
- 雲端同步、多裝置支持
- 完整交易記錄（買賣歷史）
- 公司行動處理（除權息、股票分割）
- 多筆持倉同一代號
- 自動匯入銀行/券商資料
- App Store / Google Play 上架
- 國內基金自動淨值拉取
- Flutter Web 版本
- 匯出/備份功能（Phase 7 選做）

## What already exists

- **Flutter 內建:** BottomNavigationBar, Form validation, Navigator 2.0
- **drift:** 完整的 SQLite ORM with code gen, migration support
- **Riverpod:** 狀態管理框架，含 code gen
- **fl_chart:** 圓餅圖 (PieChart)、折線圖 (LineChart) 現成元件
- **go_router:** 宣告式路由，Flutter 官方推薦
- **dio:** HTTP client with interceptors, 可以寫快取攔截器
- **uuid package:** UUID 生成

## Failure Modes

| 失敗場景 | 影響 | 測試覆蓋 | 錯誤處理 | 用戶看到什麼 |
|---------|------|---------|---------|------------|
| Yahoo Finance API 掛了 | 股價不更新 | ✓ planned | ✓ fallback + cache | 過期標示 |
| ExchangeRate-API 掛了 | 匯率不更新 | ✓ planned | ✓ 用 DB 最新 | 淡色「昨天匯率」 |
| CoinGecko API 掛了 | 幣價不更新 | ✓ planned | ✓ cache | 過期標示 |
| 手機無網路 | 全部 API 失敗 | ✓ planned | ✓ 全用快取 | 全部顯示過期 |
| DB 損壞 | 資料遺失 | ✗ not planned | ✗ 無備份 | 空 app |
| 輸入不合法值 | 資料異常 | ✓ planned | ✓ form validation | 表單錯誤提示 |

**Critical gap:** DB 損壞無備份。v1 可接受（用戶已決定備份留 Phase 7）。
